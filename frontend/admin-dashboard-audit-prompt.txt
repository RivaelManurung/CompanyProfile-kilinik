You are a Senior Principal Software Engineer, Product Designer, and Security Reviewer.

Task:
Audit the admin dashboard source code and produce a production-grade `feedback.md`.

Context:
The project is a modern web application admin dashboard, likely using Next.js, React, Tailwind, shadcn/ui, Framer Motion, and API integration. The design must not look like a generic AI-generated dashboard. It must feel professional, calm, scalable, and production-ready.

Important:
Do not give generic advice. Read the actual source code first. Mention exact file paths, components, routes, and code-level issues. If something is not present in the code, say it is missing. If something is good, mention why it is good. If something is risky, explain the production impact and how to fix it.

Output:
Create or overwrite a file named:

feedback.md

The feedback.md must contain:

1. Executive Summary
- Overall readiness percentage.
- Production status:
  - NOT READY
  - INTERNAL ALPHA
  - UAT READY
  - PRODUCTION READY
- Short summary of top risks.

2. Scorecard Table
Audit these areas with score 1-10:
- Visual Design Quality
- Admin Information Architecture
- Sidebar & Navigation UX
- Dashboard KPI Quality
- Data Table UX
- CRUD/Form UX
- RBAC & Authorization
- API Integration
- Loading/Error/Empty States
- Performance
- Accessibility
- Animation Quality
- Maintainability
- Production Readiness

For each row include:
- Score
- Status
- Core issue
- Priority

3. Detailed Findings
Group findings by severity:

P0 Critical:
- Security holes
- Frontend-only RBAC
- Exposed secrets
- Unsafe destructive actions
- Missing route protection
- Sensitive data leakage
- Broken auth/session handling

P1 High:
- Poor dashboard hierarchy
- Generic AI-looking UI
- Bad sidebar architecture
- Weak table UX
- Missing loading/error/empty states
- Bad API error handling
- No server-side pagination/filtering
- Heavy client-side rendering
- Poor form validation
- Inconsistent design system

P2 Medium:
- Minor UI inconsistencies
- Weak microcopy
- Spacing/radius/shadow inconsistency
- Animation too much or too little
- Accessibility gaps
- Component duplication

P3 Low:
- Naming polish
- Refactor suggestions
- File organization improvements
- Small visual refinements

Each finding must include:
- Title
- Severity
- Location/file path
- Problem
- Why it matters
- Recommended fix
- Example implementation if helpful

4. Anti AI-Design Audit
Specifically judge whether the dashboard looks like AI-generated template design.

Check for:
- Too many random icons
- Sparkle/wand/AI icons where not needed
- Overused gradient/glassmorphism
- Repetitive card layout
- Fake-looking metrics
- No operational hierarchy
- Generic copywriting
- Weak empty states
- Too many visual effects
- Lack of domain-specific UX

Give a verdict:
- AI-looking
- Semi-professional but still template-like
- Professional and production-grade

Then explain exactly what must change.

5. Animation Audit
Review all Framer Motion / CSS animation usage.

Check:
- Is animation purposeful?
- Does it help clarity?
- Does it respect prefers-reduced-motion?
- Are durations/easing consistent?
- Are hover effects too much?
- Are page transitions smooth?
- Are dashboard animations too landing-page-like?

Recommend:
- Animation tokens
- Standard variants
- Where animation should exist
- Where animation should be removed

6. Security Audit

- Route protection
- Middleware
- Cookie/session/token handling
- Permission checks
- API calls
- Destructive actions
- Export/download actions
- NEXT_PUBLIC env usage
- Hardcoded secrets
- Client-side role manipulation risk

Frontend permission hiding is not enough. Backend/API authorization must exist.

- Sorting

- Error state

- Double-submit prevention
- Dirty-state warning

- Accessibility labels

- heavy dependencies

- unnecessary re-renders
- large data loaded client-side
- image optimization
- bundle risk

12. Final Verdict

Quality bar:
