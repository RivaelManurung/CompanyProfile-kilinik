====================================================================
P0 SECURITY FIXES - MUST DO FIRST
====================================================================

Fix the following issues from the audit:

1. Remove demo credentials from admin login UI.
- Remove any visible admin demo email/password.
- Do not show seeded admin credentials in production UI.
- If demo/local hints are needed, guard them with NODE_ENV === "development".

2. Stop returning JWT token in the login JSON response.
- Backend should set httpOnly cookie only.
- Frontend must not receive/store/access the token.
- Update API client types accordingly.

3. Harden production config.
- Backend must fail fast when production secrets are still default.
- Validate JWT_SECRET, ADMIN_PASSWORD, DATABASE_URL, cookie secure settings, CORS origins.
- Never allow default admin password in production.

4. Add backend RBAC/permission enforcement.
- Frontend menu hiding is not security.
- Add backend middleware/policy checks for admin routes.
- Define roles/capabilities clearly:
  - super_admin
  - clinic_admin
  - receptionist
  - content_editor
  - viewer
- Protect destructive actions like delete/export/status-change.

5. Improve frontend route/session validation.
- Do not only check cookie presence.
- Add a lightweight session verification endpoint if missing.
- Admin shell should redirect expired/invalid sessions cleanly.
- Preserve backend as the final source of truth.

6. Add rate limiting.
- Add login rate limiting.
- Add public appointment creation rate limiting.
- Add sane error responses for throttled requests.

7. Add CSRF/origin protection for cookie-authenticated mutations.
- Since auth uses cookies, protect POST/PUT/PATCH/DELETE.
- Use CSRF token or strict Origin/Referer validation depending on existing architecture.

8. Add audit logs for sensitive admin actions.
- Track create/update/delete/status-change/login-failed/login-success where applicable.
- Include admin id, action, resource, resource id, timestamp, ip/user-agent if available.
